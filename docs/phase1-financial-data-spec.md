# Phase 1 財務データ仕様

## 1. 分析対象

MVPでは次の5社を対象とする。Phase 2ではZOZOの1社だけを先に取得し、取得・保存・照合の手順が固まってから残り4社へ広げる。

| 取得順 | ティッカー | 企業名 | 選定目的 |
|---:|---|---|---|
| 1 | `3092.T` | ZOZO | 1社目の取得・照合手順を確立する |
| 2 | `2432.T` | DeNA | 異なる事業構成の企業を追加する |
| 3 | `4751.T` | サイバーエージェント | 決算期が異なる企業を扱う |
| 4 | `3626.T` | TIS | 比較的安定したITサービス企業を加える |
| 5 | `4478.T` | freee | 赤字や負値を含む可能性があるデータを扱う |

対象期間は、各企業について取得時点で利用できる直近5会計年度とする。四半期報告書ではなく、有価証券報告書の年次連結データを優先する。

`fiscal_year` は会計年度が終了した暦年とする。例えば2024年4月1日から2025年3月31日までの会計年度は `2025` と記録する。正確な期末日は `fiscal_year_end` にISO 8601形式で保存する。

EDINET DBの年次財務レスポンスには各年度の正確な期末日が含まれない。取得元から確認できない間は `fiscal_year_end` を `null` とし、企業ごとの決算月から推測しない。取得元を追加した時点でISO 8601形式の日付へ置き換える。

## 2. 取得する指標

APIから取得した値は `metrics` に保存する。企業や会計基準によって名称が異なる場合も、アプリ内部では次の名前に統一する。

| 標準名 | 意味 | 単位 |
|---|---|---|
| `revenue` | 売上高または売上収益 | 円 |
| `operating_income` | 営業利益。損失は負値 | 円 |
| `net_income` | 親会社株主または親会社所有者に帰属する当期利益。損失は負値 | 円 |
| `total_assets` | 期末の総資産 | 円 |
| `equity` | 親会社株主または親会社所有者に帰属する持分 | 円 |
| `operating_cash_flow` | 営業活動によるキャッシュフロー | 円 |
| `investing_cash_flow` | 投資活動によるキャッシュフロー。支出超過は負値 | 円 |
| `roe_pct` | 自己資本利益率 | パーセントポイント |

元レスポンスに該当値がない場合、推測やゼロ補完は行わず `null` とする。ROEもMVPでは取得元の値を使用し、独自計算への置き換えは行わない。

## 3. 自前で計算する指標

計算値は `calculated_metrics` に保存する。割合は小数比率ではなくパーセントポイントで表す。例えば12.5%は `12.5` とする。

### 売上高前年比

```text
revenue_yoy_pct = (当年度売上高 / 前年度売上高 - 1) * 100
```

当年度または前年度が `null`、もしくは前年度が0の場合は `null` とする。

### 営業利益率

```text
operating_margin_pct = 営業利益 / 売上高 * 100
```

営業利益または売上高が `null`、もしくは売上高が0の場合は `null` とする。営業損失の場合は負の割合を保持する。

### フリーキャッシュフロー

```text
free_cash_flow = 営業キャッシュフロー + 投資キャッシュフロー
```

投資キャッシュフローの符号を反転させない。どちらか一方が `null` の場合は `null` とする。

### 売上高3年CAGR

```text
revenue_cagr_3y_pct = ((当年度売上高 / 3期間前の売上高) ** (1 / 3) - 1) * 100
```

「3年」は3会計期間の差を意味するため、計算には当年度と3期間前を含む最大4年度分が必要となる。開始値が0以下、終了値が負、またはいずれかが `null` の場合は `null` とする。

計算途中では丸めず、JSONへ保存するときにパーセント値を小数第4位までに丸める。画面表示用の丸めはフロントエンド側で別に行う。

## 4. 欠損値とゼロ

- `null`: 取得元に値がない、意味を確定できない、または計算条件を満たさない
- `0`: 取得元の値または計算結果が実際にゼロ
- 負値: 損失やキャッシュ流出としてそのまま保持する
- 空文字、`"N/A"`、ハイフンを数値フィールドへ保存しない

フロントエンドでは `null` を `—` と表示し、ゼロは `0` と表示する。

## 5. 単位と精度

- 金額はすべて整数の円へ正規化する
- 通貨コードは `JPY` とする
- 億円や兆円への変換は表示時だけ行う
- 割合はパーセントポイントで保存する
- APIの元単位と正規化処理は変換ログで追跡できるようにする

## 7. 列挙値と並び順

- `accounting_standard`: `JGAAP`、`IFRS`、`USGAAP`、`UNKNOWN`
- `document_type`: MVPでは `annual_securities_report` のみ
- `periods`: `fiscal_year` の昇順。同一年度内の並び替えが必要になった場合のみ `fiscal_year_end` を使用する
- `companies`: Phase 1の取得順
- 日時: UTCのISO 8601形式（末尾 `Z`）

## 8. Phase 1完了判定

- 対象5社と最初に取得する1社が決まっている
- 取得値と計算値が別オブジェクトになっている
- 全指標の意味と単位が決まっている
- `null`、ゼロ、負値の扱いが決まっている
- 3年CAGRに必要な期間数が明示されている
- 出典書類と取得日時を年度ごとに追跡できる
- 画面用JSONのトップレベルから年度別データまで説明できる
